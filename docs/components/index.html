Buttons #

Alle knapper følger egen standard, primary, secondary og call-to-action. Der er kjøreregler på når hver enkelt skal brukes

Generelt

For semantisk struktur burde knapper være markert som button, alternativt a-tags

All styling på knapper styres av klasser. Du finner klassene under:

Lexto - Frontend\css\_modules\_buttons.scss

Alle knapper skal ha hovedklassen .btn som gir generell styling som padding, margin osv. Men må kombineres med hvilken type knapp det er, for eksempel:

                            Markup
                            <button class="btn btn-primary">Click me!</button>
                        
                            Preview

Call-to-action buttons

  • CTA knapper skal det kun være en av per side, brukes kun for å visuelt guide bruker mot å gjøre en aksjon
  • Styres ved å gi klassen btn btn-cta

Primary buttons

  • Primary knapper skal ideelt sett kun brukes 1-3 ganger på samme side
  • Skal være visuell guide for bruker for å komme dit brukeren skal
  • Styres ved å gi klassen btn btn-primary

Secondary Buttons

  • Kan brukes 'over alt' for sekundære handlinger tilgjengelige for brukeren.
  • Brukes eksempelvis på avbryt knapper eller slett knapper
  • Styres ved å gi klassen btn btn-secondary

Card #

Kort brukes mye rundt om på systemet. Ofte brukes de på steg der du oppretter noe, f.eks en ny bruker eller kontakt, men også nyhetsartikler o.l.

Verdier .card klassen gir:

  • 1rem padding (for spacing)
  • 5px border radius
  • default box-shadow (for bedre synlighet)

Eksempel

                            Markup
                            <div class="card">I'm a card!</button>
                        
                            Preview
                            
I'm a card!

Container #

Brukes for generell wrapping av innhold på alle sider. Setter margin/padding for spacing samt styling for å gi en uthevet effekt.

Stylingen for kontainere finner du under:

Lexto - Frontend\css\_molecules\_mol_container.scss

Verdier .mol_container klassen gir:

  • Padding og margin på 1rem
  • Default box-shadow
  • 5px border-radius
  • Relativ posisjon (for absolutte elementer)
  • Bakgrunnsfarge: #fefefe

Eksempel

                            Markup
                            <div class="mol_container">I'm a container!!!</button>
                        
                            Preview
                            
I'm a container!!!
